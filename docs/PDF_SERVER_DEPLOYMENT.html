🖨️ PDF Server Deployment Guide

Deploy Your PDF Server to Keep It Always Running

🚨 Current Issue

Problem: Your PDF server is only running locally. When users visit your live app at https://scs-lims.web.app, they can't generate PDFs because the PDF server isn't accessible.

Solution: Deploy the PDF server to a cloud hosting service that keeps it running 24/7.

🔥 Option 1: Render.com (Recommended)

✅ Best For: Free hosting with always-on service
📊 Difficulty: Easy
💰 Cost: FREE (with limitations)
⚡ Uptime: 24/7 (sleeps after 15 min inactivity, wakes up on request)

Step-by-Step Deployment:

  1. Create Render.com Account

    Go to https://render.com and sign up (free)

  2. Connect Your GitHub Repository

    In Render dashboard, click "New +" → "Web Service"

    Connect your GitHub account and select your LIMS repository

  3. Configure the Service
    Name: lims-pdf-server Root Directory: server Environment: Node Build Command: npm install Start Command: node index.js Instance Type: Free
  4. Deploy

    Click "Create Web Service" and wait for deployment (2-3 minutes)

    Your PDF server will be available at: https://lims-pdf-server.onrender.com

  5. Update Frontend

    Edit src/services/pdfService.ts and change:

    // Change from: const PDF_SERVER_URL = 'http://localhost:3001'; // To: const PDF_SERVER_URL = 'https://lims-pdf-server.onrender.com';
  6. Redeploy Frontend
    npm run build firebase deploy --only hosting
📝 Note: Free tier sleeps after 15 minutes of inactivity but wakes up automatically when needed (may take 30-60 seconds to wake up).

Step 1: Update PDF Service

Edit src/services/pdfService.ts:

// Find this line (around line 10): const PDF_SERVER_URL = 'http://localhost:3001'; // Replace with your deployed server URL: const PDF_SERVER_URL = 'https://lims-pdf-server.onrender.com'; // OR const PDF_SERVER_URL = 'https://your-app.railway.app';

Visit your live app and try generating a PDF. It should work now!

🔧 Troubleshooting

Common Issues:

❌ PDF Server Not Responding

  • Check if the server URL is correct in pdfService.ts
  • Verify the server is deployed and running
  • Test the server directly: https://your-server-url.onrender.com/health
  • Check Render/Railway logs for errors

❌ CORS Errors

  • Make sure your PDF server has CORS enabled
  • Check that your frontend domain is allowed
  • Verify the server is accessible from your frontend

❌ Slow PDF Generation

  • Free tier servers "sleep" after inactivity
  • First request after sleep takes 30-60 seconds
  • Subsequent requests are fast
  • Consider upgrading to paid plan for always-on service

❌ Build Failures

  • Check that package.json exists in server folder
  • Verify all dependencies are listed
  • Check build logs in Render/Railway dashboard
  • Ensure Node.js version compatibility

⚡ Performance Tips

For Free Tier:

  • ✅ First request after sleep: 30-60 seconds
  • ✅ Subsequent requests: 2-5 seconds
  • ✅ Server sleeps after 15 minutes of inactivity
  • ✅ Wakes up automatically on request

For Better Performance:

  • 💡 Upgrade to paid plan for always-on service
  • 💡 Use a "ping" service to keep server awake
  • 💡 Consider caching frequently generated PDFs

Keep Server Awake (Optional):

You can use a service like UptimeRobot to ping your server every 10 minutes:

# Add this to your server/index.js app.get('/ping', (req, res) => { res.json({ status: 'alive', timestamp: new Date().toISOString() }); });

Then set up UptimeRobot to ping: https://your-server.onrender.com/ping
