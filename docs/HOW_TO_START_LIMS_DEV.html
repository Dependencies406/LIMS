Complete Guide for Developers

Do NOT just use npm run dev!

That's exactly what caused your PDF generation problem. Your LIMS app is different from normal React apps.

🔍 Understanding the Problem

Normal React App vs Your LIMS App

Type Command Result
Normal React App npm run dev ✅ Works perfectly (one server)
Your LIMS App npm run dev ❌ NOT ENOUGH (missing PDF server!)

🏗️ Your App Architecture

Your LIMS has TWO separate servers that MUST both run:

┌─────────────────────────────────────────┐ │ YOUR LIMS APPLICATION │ ├─────────────────────────────────────────┤ │ │ │ ┌────────────────────────────────┐ │ │ │ FRONTEND (React + Vite) │ │ │ │ Port: 5173 │ │ │ │ Started by: npm run dev │ │ │ │ Does: UI, Forms, Display │ │ │ └────────────────────────────────┘ │ │ ↓ ↑ │ │ (needs to call) │ │ ↓ ↑ │ │ ┌────────────────────────────────┐ │ │ │ PDF SERVER (Node + Puppeteer) │ │ │ │ Port: 3001 │ │ │ │ Started by: node server/index │ │ │ │ Does: Generate PDFs │ │ │ └────────────────────────────────┘ │ │ │ └─────────────────────────────────────────┘

✅ Both servers are required for full functionality!

  • Frontend handles the user interface
  • PDF Server generates PDFs with Thai font support
  • They work together as one application

✅ CORRECT Way to Start Development

Option 1: Using the Batch File (EASIEST) ⭐

Just double-click this file: START_HERE.bat

What it does:

  • Automatically kills old processes
  • Starts PDF Server on port 3001
  • Starts Frontend on port 5173
  • Opens 2 separate windows
  • You're ready to code!

Option 2: Manual Start (if batch file fails)

Terminal 1 - Start PDF Server:

cd server
node index.js

Terminal 2 - Start Frontend:

npm run dev

Keep both terminals open while working!

❌ WRONG Way (What You Were Doing)

npm run dev  # ❌ Only starts frontend
             # ❌ PDF server not started
             # ❌ PDF generation will fail

Result:

  • Frontend works ✅
  • Jobs/Customers work ✅
  • PDF generation FAILS ❌ ← This was your error!

💡 Why Two Servers?

Why Not Just Frontend?

Because your app needs:

  • Thai Font Support - Puppeteer handles Thai characters properly
  • Complex PDF Layouts - Server-side rendering for accurate PDFs
  • Heavy Processing - Puppeteer is too heavy for browser

Could We Combine Them?

Technically yes, but it would:

  • Make the app slower
  • Complicate deployment
  • Reduce flexibility
  • Break Thai fonts

Current architecture is correct and professional.

🌟 The Golden Rule 🌟

ALWAYS USE START_HERE.bat
NEVER USE npm run dev ALONE

❓ Frequently Asked Questions

Q: Can I just use npm run dev?

A: ❌ NO! You must use START_HERE.bat or manually start both servers.

Q: Why not?

A: Because npm run dev only starts the frontend. PDF generation needs the PDF server on port 3001.

Q: What if I forget?

A: The yellow warning banner will remind you immediately!

Q: Is START_HERE.bat safe?

A: Yes! It just starts both servers. You can open it in a text editor to see exactly what it does.

Q: Can I use VS Code terminal?

A: Yes, but you need 2 terminals:

# Terminal 1
cd server && node index.js

# Terminal 2  
npm run dev
