📄 PDF Generation Alternatives for LIMS

Instead of running your own PDF server with Puppeteer, here are better alternatives that are easier to maintain and more reliable.

🏆 Option 1: Client-Side PDF Generation (RECOMMENDED)

Use jsPDF + html2canvas

Easy Free No Server Needed

Description: Generate PDFs directly in the browser using JavaScript libraries. No server required!

✅ Pros:

  • 100% free - no hosting costs
  • No server maintenance
  • Works offline
  • Fast - no network latency
  • Easy to implement
  • Complete control over PDF generation

❌ Cons:

  • Slightly lower quality than Puppeteer
  • Limited CSS support
  • Runs on user's device (slower on old devices)

Implementation:

// Install packages
npm install jspdf html2canvas

// src/services/clientPdfService.ts
import jsPDF from 'jspdf';
import html2canvas from 'html2canvas';

export const generateJobPDF = async (job: Job): Promise<void> => {
  // Create a temporary div with your job content
  const element = document.createElement('div');
  element.innerHTML = `
    <div style="padding: 20px; font-family: Arial;">
      <h1>Job Report</h1>
      <p><strong>Job ID:</strong> ${job.id}</p>
      <p><strong>Title:</strong> ${job.title}</p>
      <p><strong>Status:</strong> ${job.status}</p>
      <!-- Add more job details -->
    </div>
  `;
  
  document.body.appendChild(element);
  
  // Convert to canvas
  const canvas = await html2canvas(element, {
    scale: 2,
    useCORS: true,
    logging: false
  });
  
  // Remove temporary element
  document.body.removeChild(element);
  
  // Create PDF
  const imgData = canvas.toDataURL('image/png');
  const pdf = new jsPDF('p', 'mm', 'a4');
  const imgWidth = 210; // A4 width in mm
  const imgHeight = (canvas.height * imgWidth) / canvas.width;
  
  pdf.addImage(imgData, 'PNG', 0, 0, imgWidth, imgHeight);
  pdf.save(`job-${job.id}.pdf`);
};

💰 Cost:

FREE - No ongoing costs, just npm packages

Option 2: pdfmake (Pure JavaScript PDF)

Use pdfmake

Easy Free

Description: Create PDFs using a declarative JavaScript API. Better quality than jsPDF.

✅ Pros:

  • 100% free
  • Better quality than jsPDF
  • No server needed
  • Great for structured documents
  • Supports tables, images, headers/footers

❌ Cons:

  • Different API - need to learn document definition
  • Can't directly convert HTML
  • More code to write

Implementation:

// Install
npm install pdfmake

// Usage
import pdfMake from 'pdfmake/build/pdfmake';
import pdfFonts from 'pdfmake/build/vfs_fonts';
pdfMake.vfs = pdfFonts.pdfMake.vfs;

export const generateJobPDF = (job: Job): void => {
  const docDefinition = {
    content: [
      { text: 'Job Report', style: 'header' },
      { text: `Job ID: ${job.id}`, margin: [0, 10, 0, 5] },
      { text: `Title: ${job.title}`, margin: [0, 0, 0, 5] },
      { text: `Status: ${job.status}`, margin: [0, 0, 0, 5] },
      // Add more content
    ],
    styles: {
      header: {
        fontSize: 18,
        bold: true,
        margin: [0, 0, 0, 10]
      }
    }
  };
  
  pdfMake.createPdf(docDefinition).download(`job-${job.id}.pdf`);
};

💰 Cost:

FREE

Option 3: Cloud PDF Services

Use PDFShift, DocRaptor, or similar

Very Easy Paid

Description: Third-party APIs that convert HTML to PDF in the cloud.

✅ Pros:

  • Highest quality (uses real Chrome)
  • No server maintenance
  • Reliable and fast
  • Easy to implement
  • Supports full CSS/HTML

❌ Cons:

  • Costs money (pay per PDF)
  • Depends on third-party service
  • Privacy concerns (sending data to third party)

Popular Services:

  • PDFShift: $9/month for 250 PDFs, then $0.01/PDF
  • DocRaptor: $15/month for 125 PDFs, then $0.10/PDF
  • API2PDF: $9/month for 1000 PDFs

💰 Cost:

$9-15/month + usage fees

Option 4: Firebase Cloud Functions

Use Firebase Functions with Puppeteer

Medium Paid (but cheap)

Description: Run Puppeteer in Firebase Cloud Functions - serverless, only pay when used.

✅ Pros:

  • Serverless - no server to maintain
  • Only pay when PDFs are generated
  • Integrates with your existing Firebase
  • High quality (real Chrome)
  • Auto-scales

❌ Cons:

  • Requires Firebase Blaze (pay-as-you-go) plan
  • Cold start delays (first PDF is slow)
  • More complex setup
  • Limited to 9 minutes execution time

💰 Cost:

~$0.01 per PDF (very cheap for low usage)

Option 5: React-PDF

Use @react-pdf/renderer

Medium Free

Description: Create PDFs using React components - very elegant solution.

✅ Pros:

  • 100% free
  • Write PDFs using React syntax
  • Good quality
  • No server needed
  • Type-safe with TypeScript

❌ Cons:

  • Need to rewrite your HTML as React components
  • Different styling approach
  • Learning curve

Implementation:

// Install
npm install @react-pdf/renderer

// Create PDF component
import { Document, Page, Text, View, StyleSheet } from '@react-pdf/renderer';

const styles = StyleSheet.create({
  page: { padding: 30 },
  section: { margin: 10, padding: 10 }
});

const JobPDF = ({ job }) => (
  <Document>
    <Page size="A4" style={styles.page}>
      <View style={styles.section}>
        <Text>Job Report</Text>
        <Text>Job ID: {job.id}</Text>
        <Text>Title: {job.title}</Text>
      </View>
    </Page>
  </Document>
);

// Generate and download
import { pdf } from '@react-pdf/renderer';

const generatePDF = async (job: Job) => {
  const blob = await pdf(<JobPDF job={job} />).toBlob();
  const url = URL.createObjectURL(blob);
  const link = document.createElement('a');
  link.href = url;
  link.download = `job-${job.id}.pdf`;
  link.click();
};
