🚀 LIMS Deployment Guide

How to Deploy Your Laboratory Information Management System

🔥 Option 1: Firebase Hosting (Recommended)

✅ Best For: Your app (already using Firebase)
📊 Difficulty: Easy
💰 Cost: Free tier available
⚡ Speed: Global CDN, very fast
  1. Install Firebase CLI
    npm install -g firebase-tools
  2. Login to Firebase
    firebase login

    This will open your browser to authenticate

  3. Initialize Firebase Hosting
    firebase init hosting

    Answer the prompts:

    • ✅ Use existing project (select your LIMS project)
    • ✅ Public directory: dist
    • ✅ Configure as single-page app: Yes
    • ✅ Overwrite index.html: No
  4. Build Your App
    npm run build

    This creates an optimized production build in the dist folder

  5. Deploy to Firebase
    firebase deploy --only hosting

    Your app will be live at: https://your-project-id.web.app

📝 Note: The PDF server (Node.js) needs to be deployed separately! See Option 4 below.

⚡ Option 2: Vercel

✅ Best For: Fast deployment with Git integration
📊 Difficulty: Very Easy
💰 Cost: Free tier available
⚡ Speed: Edge network, instant deploys
  1. Install Vercel CLI
    npm install -g vercel
  2. Login to Vercel
    vercel login
  3. Deploy
    vercel

    Follow the prompts. Vercel will automatically detect Vite and configure everything!

  4. Production Deploy
    vercel --prod

    Your app will be live at: https://your-app.vercel.app

⚠️ Important: You'll need to set environment variables in Vercel dashboard!

🌐 Option 3: Netlify

✅ Best For: Simple drag-and-drop deployment
📊 Difficulty: Very Easy
💰 Cost: Free tier available
⚡ Speed: Global CDN
  1. Build Your App
    npm run build
  2. Go to Netlify

    Visit https://netlify.com

  3. Drag & Drop

    Drag your dist folder to Netlify's deployment zone

    OR use CLI:

    npm install -g netlify-cli netlify deploy --prod

🖨️ Option 4: Deploy PDF Server (Required!)

⚠️ Critical: Your PDF generation server needs to be deployed separately from the frontend!

Recommended: Render.com (Free Tier)

  1. Create account at Render.com

    Visit https://render.com

  2. Create New Web Service
    • Connect your GitHub repository
    • Root directory: server
    • Build command: npm install
    • Start command: node index.js
    • Environment: Node
  3. Update Frontend PDF_SERVER_URL

    In src/services/pdfService.ts, change:

    const PDF_SERVER_URL = 'https://your-pdf-server.onrender.com';

    Replace with your Render.com URL

  4. Rebuild and Redeploy Frontend
    npm run build firebase deploy --only hosting

Alternative: Railway.app

  • Similar to Render.com
  • Free tier available
  • Automatic deployments from Git

Alternative: Heroku

  • Classic platform
  • No free tier anymore (paid only)
  • Very reliable

🔐 Environment Variables Setup

⚠️ Important: Never commit your .env file! Use platform-specific environment variable settings.

For Firebase Hosting:

Environment variables are built into the app at build time:

# Your .env file (already have this) VITE_FIREBASE_API_KEY=your-api-key VITE_FIREBASE_AUTH_DOMAIN=your-app.firebaseapp.com VITE_FIREBASE_PROJECT_ID=your-project-id # ... etc

For Vercel:

  1. Go to your project dashboard
  2. Settings → Environment Variables
  3. Add each VITE_* variable from your .env
  4. Redeploy

For Netlify:

  1. Site settings → Environment variables
  2. Add each variable
  3. Trigger new deploy

📋 Complete Deployment Workflow

Step-by-Step Full Deployment:

1️⃣ Deploy Frontend (Firebase Hosting)

# Install Firebase CLI (one-time) npm install -g firebase-tools # Login to Firebase (one-time) firebase login # Initialize hosting (one-time) firebase init hosting # Choose: # - Use existing project # - Public directory: dist # - Single-page app: Yes # - Overwrite index.html: No # Build the app npm run build # Deploy to Firebase firebase deploy --only hosting # Your app will be live at: # https://your-project-id.web.app

2️⃣ Deploy PDF Server (Render.com)

# Option A: Connect via Render.com Dashboard # 1. Go to https://render.com # 2. Click "New +" → "Web Service" # 3. Connect your GitHub repo # 4. Configure: # - Name: lims-pdf-server # - Root Directory: server # - Build Command: npm install # - Start Command: node index.js # - Instance Type: Free # Option B: Use render.yaml (in your repo root) # Create render.yaml: services: - type: web name: lims-pdf-server env: node rootDir: server buildCommand: npm install startCommand: node index.js plan: free healthCheckPath: /health

3️⃣ Update Frontend to Use Production PDF Server

Edit src/services/pdfService.ts:

// Change from: const PDF_SERVER_URL = 'http://localhost:3001'; // To: const PDF_SERVER_URL = import.meta.env.VITE_PDF_SERVER_URL || 'https://your-pdf-server.onrender.com';

Add to your .env:

VITE_PDF_SERVER_URL=https://your-pdf-server.onrender.com

Then rebuild and redeploy:

npm run build firebase deploy --only hosting

🌐 Custom Domain Setup (Optional)

For Firebase Hosting:

  1. Go to Firebase Console

    Hosting → Add custom domain

  2. Add Your Domain

    Enter your domain (e.g., lims.yourcompany.com)

  3. Verify Domain

    Add TXT record to your DNS provider

  4. Configure DNS

    Add A records pointing to Firebase's IP addresses

  5. Wait for SSL

    Firebase automatically provisions SSL certificate (can take up to 24 hours)

🔧 Troubleshooting

Common Issues:

❌ Build Fails

  • Check TypeScript errors: npm run build
  • Verify all dependencies installed: npm install
  • Check for linting errors

❌ Environment Variables Not Working

  • Make sure variables start with VITE_
  • Rebuild after changing .env: npm run build
  • Check platform-specific env var settings

❌ PDF Generation Fails

  • Verify PDF server is deployed and running
  • Check PDF_SERVER_URL points to production server
  • Ensure PDF server has /health endpoint working
  • Check CORS settings on PDF server

❌ Firebase Auth Not Working

  • Add your domain to Firebase Console → Authentication → Authorized domains
  • Update Firebase project settings

🔄 Continuous Deployment (Advanced)

Automatic Deployment from Git

For automatic deployments when you push code:

With Vercel (Easiest):

  1. Connect your GitHub repository
  2. Every push to main branch auto-deploys
  3. Pull requests get preview deployments

With Firebase + GitHub Actions:

Create .github/workflows/deploy.yml:

name: Deploy to Firebase on: push: branches: [ main ] jobs: deploy: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - uses: actions/setup-node@v2 with: node-version: '18' - run: npm install - run: npm run build - uses: FirebaseExtended/action-hosting-deploy@v0 with: repoToken: '${{ secrets.GITHUB_TOKEN }}' firebaseServiceAccount: '${{ secrets.FIREBASE_SERVICE_ACCOUNT }}' channelId: live projectId: your-project-id
